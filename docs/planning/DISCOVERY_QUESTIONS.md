# Discovery Questions

Answer these before generating project scaffolding or writing major code.

## Product Scope

1. What problem does the product solve?
2. Who are the target users?
3. What are the must-have features for MVP?
4. What is explicitly out of scope for v1?

## Platform and Delivery

5. Project type: web, mobile, desktop, API, fullstack?
6. Preferred stack and known constraints?
7. Target environments: local, staging, production?
8. Required integrations (payments, auth providers, email, analytics)?

## Security and Compliance

9. What data is sensitive (PII, tokens, health, financial)?
10. Required compliance baseline (GDPR, SOC2, internal policy)?
11. Authentication/authorization model requirements?
12. Encryption, retention, and audit requirements?

## Quality and Operations

13. Required test levels (unit, integration, e2e) and coverage target?
14. Performance and availability targets?
15. Observability requirements (logs, traces, alerts)?
16. Release cadence and rollback strategy?

## Team and Workflow

17. Branching model and PR policy?
18. Definition of Done additions for this project?
19. Documentation depth expected?
20. Any hard deadlines or milestone dates?
