# Task Generation Playbook

Use this playbook to convert discovery answers into an executable MVP backlog.

## Step 1: Convert Goals to Capabilities

- List 3-7 core capabilities needed for MVP
- Assign each capability to one module/domain

## Step 2: Slice by Vertical Value

- Create tasks that deliver user-visible value end-to-end
- Avoid horizontal-only tasks unless they remove hard blockers

## Step 3: Add Acceptance and Risk

Each task must include:
- acceptance criteria
- test plan
- security note
- risk note

## Step 4: Prioritize

Prioritize by:
1) user value
2) risk reduction
3) dependency order
