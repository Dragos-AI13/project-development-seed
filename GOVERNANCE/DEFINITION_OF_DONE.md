# Definition of Done

A task is done only when all points below are true:

- Scope implemented according to acceptance criteria
- Unit/integration tests added or updated and passing
- Coverage target from `docs/planning/PROJECT_PROFILE.yaml` is met (or approved exception documented)
- Lint/format/static checks passing
- Security checks reviewed for new attack surface
- Observability impact reviewed (logs/metrics/alerts for critical flows)
- Documentation updated (README/API/ADR if needed)
- Risk register updated when high-impact changes are introduced
- `docs/state/PROJECT_STATUS.md` updated
- `docs/state/NEXT_ACTIONS.md` updated
- `docs/state/CHANGELOG_WORKING.md` updated
- If no tests were added, a written test gap justification exists
- Code review completed and approved
