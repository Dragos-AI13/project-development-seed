# Definition of Architecture Done

An architecture change or proposal is done only when:

- Problem statement and constraints are explicit
- `docs/architecture/TECH_DECISION_MATRIX.md` is completed
- Decision is documented as ADR in `DECISIONS/`
- Security impact and controls are specified
- Data/migration impact is documented
- Observability impact (logs/metrics/alerts) is defined
- Rollback or fallback strategy is defined
- Resulting module boundaries are explicit and testable
