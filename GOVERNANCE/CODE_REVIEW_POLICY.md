# Code Review Policy

## Reviewer Focus

1. Correctness and behavioral regressions
2. Security risks and data exposure
3. Test completeness and edge coverage
4. Maintainability and architecture consistency

## Review Checklist

- Does code follow module boundaries and SRP?
- Are validation and authorization checks present?
- Are errors handled with useful context?
- Are tests meaningful and not only happy-path?
- Are docs or ADR updates required?
